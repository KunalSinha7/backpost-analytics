# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

### Full stack (Docker Compose — recommended for development)
```bash
docker compose watch          # start all services with live reload
docker compose logs backend   # tail a specific service
```

### Backend (local, without Docker)
```bash
cd backend
uv sync                       # install dependencies
uv run fastapi dev app/main.py  # run dev server at http://localhost:8000
bash ../scripts/test.sh       # run tests with coverage
```

Run a subset of backend tests (via Docker):
```bash
docker compose exec backend bash scripts/tests-start.sh -x             # stop on first failure
docker compose exec backend bash scripts/tests-start.sh tests/api/test_items.py  # single file
```

### Frontend (local, without Docker)
```bash
cd frontend
bun install                   # required before first run — node_modules not pre-installed
# Create frontend/.env pointing at the local backend (required, or login will fail):
echo "VITE_API_URL=http://localhost:8000" > .env
bun run dev                   # dev server at http://localhost:5173
bun run lint                  # biome check + auto-fix
bunx playwright test          # E2E tests (requires backend running)
bunx playwright test --ui     # interactive Playwright UI
```

### Linting (pre-commit / prek)
```bash
# From backend/ directory:
uv run prek run --all-files   # runs ruff, ruff-format, biome, yaml/toml checks
uv run prek install -f        # install git pre-commit hook
```

### Generate frontend API client
Run this any time the backend OpenAPI schema changes:
```bash
bash ./scripts/generate-client.sh
```

### Database migrations (Alembic — run inside backend container)
```bash
docker compose exec backend bash
alembic revision --autogenerate -m "Add column X to Y"
alembic upgrade head
```

## Architecture

### Backend (`backend/app/`)
- `main.py` — FastAPI app init, CORS middleware, Sentry, mounts `api_router` at `/api/v1`
- `api/main.py` — aggregates all route modules; `private.py` routes only available in `local` environment
- `api/routes/` — one file per resource: `login`, `users`, `items`, `utils`, `private`
- `api/deps.py` — FastAPI dependency injection (DB session, current user, superuser guard)
- `models.py` — all SQLModel models and Pydantic schemas in one file; follows the pattern `XBase → XCreate/XUpdate → X (table=True) → XPublic`
- `crud.py` — all database CRUD operations; call these from route handlers, not raw SQL
- `core/config.py` — `pydantic-settings` config; reads from top-level `.env`
- `core/db.py` — SQLAlchemy engine setup and `init_db()` (seeds first superuser)
- `alembic/` — migration scripts; always create a revision after changing `models.py`

### Frontend (`frontend/src/`)
- `client/` — **auto-generated** OpenAPI client; never edit manually, regenerate via `generate-client.sh`
- `routes/` — TanStack Router file-based routing; `_layout.tsx` is the authenticated shell (redirects to `/login` if not logged in); pages live under `_layout/`
- `components/` — shared React components (shadcn/ui based)
- `hooks/` — custom hooks including `useAuth` (exposes `isLoggedIn`, current user)
- `routeTree.gen.ts` — auto-generated by TanStack Router plugin; do not edit

### Data flow
Frontend (`TanStack Query` + generated client) → `GET /api/v1/...` → FastAPI route → `crud.py` → SQLModel/PostgreSQL

### Authentication
JWT tokens. The `/_layout` route group enforces auth client-side; the backend enforces it via `deps.get_current_user` / `deps.get_current_active_superuser`.

### Docker Compose files
- `compose.yml` — base config for all services
- `compose.override.yml` — development overrides (volume mounts, hot reload, local Traefik)
- `compose.traefik.yml` — production/staging Traefik config

## Key conventions

- Add new API endpoints by creating/editing a file in `backend/app/api/routes/`, then registering it in `backend/app/api/main.py`
- Add new DB tables/columns by editing `backend/app/models.py`, then running an Alembic migration
- After any backend model or route change that affects the OpenAPI schema, regenerate the frontend client
- Email templates are MJML source in `backend/app/email-templates/src/`; compile to HTML with the VS Code MJML extension and save to `build/`
- API is versioned at `/api/v1`; the `private` router is for internal/test use and is excluded in non-local environments
